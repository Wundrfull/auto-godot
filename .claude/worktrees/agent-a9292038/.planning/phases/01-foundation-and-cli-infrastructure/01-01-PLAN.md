    **3. Create src/gdauto/errors.py** (per D-15, CLI-05):
    Define `GdautoError` as a base exception dataclass with fields: `message: str`, `code: str`, `fix: str | None = None`. Add `to_dict()` method returning `{"error": message, "code": code}` plus optional `"fix"` key. Subclass into: `ParseError`, `FileNotFoundError` (shadow builtins intentionally, use `GdautoFileNotFoundError` if naming conflicts arise; better name: `ResourceNotFoundError`), `GodotBinaryError`, `ValidationError`, `ProjectError`. Each subclass is a simple pass-through; the base class carries all logic. Do NOT inherit from builtins.FileNotFoundError; keep the hierarchy clean under GdautoError.

    **4. Create src/gdauto/output.py** (per D-13, D-14, D-16, D-18, D-19):
    Define `GlobalConfig` dataclass with fields: `json_mode: bool = False`, `verbose: bool = False`, `quiet: bool = False`, `godot_path: str | None = None`.
    Define `emit(data: dict, human_fn: Callable, ctx: click.Context) -> None` that checks `ctx.obj.json_mode`: if True, outputs `json.dumps(data, indent=2)` to stdout; if False, calls `human_fn(data, verbose=ctx.obj.verbose)` unless `ctx.obj.quiet` is True.
    Define `emit_error(error: GdautoError, ctx: click.Context) -> None` that in json_mode outputs `json.dumps(error.to_dict())` to stderr; in human mode outputs "Error: {message}" in red to stderr, and if fix is present, outputs "Fix: {fix}" in yellow to stderr. Always calls `ctx.exit(1)`.

    **5. Create src/gdauto/cli.py** (per D-16, D-17, D-18, D-19, CLI-01, CLI-02, CLI-03):
    Import `rich_click as click` (drop-in replacement per D-13).
    Define root group with `@click.group(invoke_without_command=True)` and options:
    - `-j/--json` with param name `json_mode`, is_flag=True
    - `-v/--verbose`, is_flag=True
    - `-q/--quiet`, is_flag=True
    - `--no-color`, is_flag=True (rich-click handles NO_COLOR env natively per D-19)
    - `--godot-path`, type=click.Path(), envvar="GODOT_PATH" (per D-17)
    - `@click.version_option(package_name="gdauto")`
    Store `GlobalConfig` in `ctx.obj`. If `no_color` is True, set environment variable `NO_COLOR=1` so rich-click respects it.
    If `ctx.invoked_subcommand is None`, echo help.

    Register six command groups (per CLI-01):
    - `project`: "Manage Godot projects (info, validate, create)"
    - `resource`: "Inspect and manipulate Godot resources (.tres, .tscn)"
    - `export`: "Build and export Godot projects"
    - `sprite`: "Sprite sheet and SpriteFrames tools"
    - `tileset`: "TileSet creation and terrain automation"
    - `scene`: "Scene file operations"

    Each group is defined in its own file under commands/ as a `@click.group(invoke_without_command=True)` that prints help when called without subcommand. For Phase 1, export/sprite/tileset/scene are stubs with just the group decorator and help text. project and resource will get real subcommands in Plan 05.

    Add a centralized error handler by wrapping the cli group with a try/except in a `main()` function, or use Click's `standalone_mode=False` pattern. The error handler catches `GdautoError`, checks `ctx.obj.json_mode`, and outputs accordingly via `emit_error`.

Import `rich_click as click`. project.py and resource.py will be fleshed out in Plan 05; the others are stubs for this phase.
