    **Create src/gdauto/formats/uid.py** (per FMT-05, research UID algorithm):

    Constants:
    ```python
    CHARS = "abcdefghijklmnopqrstuvwxy012345678"  # 34 chars: a-y, 0-8 (no z, no 9)
    BASE = 34
    MAX_UID = (1 << 63) - 1  # 63-bit max
    RESOURCE_ID_CHARS = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789_"
    RESOURCE_ID_LENGTH = 5
    ```

    Functions:
    - `generate_uid() -> int`: Use `secrets.randbelow(MAX_UID + 1)` to generate a 63-bit random value.
    - `uid_to_text(uid: int) -> str`: Convert to base-34 encoding using CHARS. Encode least-significant digit first (matching Godot's algorithm). Return `"uid://" + encoded`.
    - `text_to_uid(text: str) -> int`: Decode uid:// text. Return -1 on invalid input. Decode most-significant digit first (reverse of encoding).
    - `generate_resource_id(type_name: str) -> str`: Generate `Type_xxxxx` format ID using `secrets.choice(RESOURCE_ID_CHARS)` for 5 characters. Return `f"{type_name}_{suffix}"`.
    - `write_uid_file(path: Path, uid_text: str) -> None`: Write uid_text + newline to `path.uid`.
    - `read_uid_file(path: Path) -> str | None`: Read `.uid` companion file, return stripped content or None.

    **Create src/gdauto/formats/common.py** (shared parsing, per research Pattern 4):

    Data models:
    ```python
    @dataclass
    class HeaderAttributes:
        """Parsed attributes from a bracket section header."""
        tag: str  # e.g., "gd_resource", "ext_resource", "node"
        attrs: dict[str, str]  # key=value pairs from header

    @dataclass
    class Section:
        """A parsed section from a .tres/.tscn file."""
        header: HeaderAttributes
        properties: list[tuple[str, Any]]  # Ordered list of (key, parsed_value)
        raw_properties: list[tuple[str, str]]  # Ordered list of (key, raw_value_string) for round-trip
        leading_whitespace: list[str]  # Blank lines and comments before this section
        trailing_comment: str | None  # Inline comment on a property line
    ```

    Parsing functions:
    - `parse_section_header(line: str) -> HeaderAttributes | None`: Parse a `[tag key=value ...]` line. Handle quoted values (`type="SpriteFrames"`), unquoted values (`format=3`), and bare tags (`[resource]`). Return None if line is not a section header.

    - `parse_sections(text: str) -> tuple[HeaderAttributes, list[Section]]`: The main parser. Line-by-line state machine with states:
      - `IDLE`: Looking for section headers or file header. Accumulate blank lines and comments.
      - `PROPERTIES`: Inside a section, reading key=value pairs.
      - `MULTILINE`: Accumulating a multi-line value (bracket depth > 0).

      For each line:
      1. If empty/blank: record as whitespace for next section.
      2. If comment (starts with `;`): record as whitespace/comment for next section.
      3. If section header (`[tag ...]`): finalize current section, start new section, parse header.
      4. If `key = value`: parse property. Check bracket balance of value; if unbalanced, enter MULTILINE state and accumulate lines until balanced.
      5. In MULTILINE state: append line to buffer, check bracket balance of accumulated text. When balanced, finalize property.

      Bracket depth tracking: increment on `{`, `[`, `(`; decrement on `}`, `]`, `)`. Skip characters inside double-quoted strings (track `in_string` state, handle `\"` escapes). This prevents false matches on brackets inside string literals (Pitfall 3 from research).

      Return the file header and list of sections.

    - `serialize_sections(header: HeaderAttributes, sections: list[Section]) -> str`: Reconstruct the file from header + sections. For each section, output leading whitespace, then the header line, then properties using raw_properties strings (for round-trip fidelity per D-05). If a property was modified (raw_properties updated), use serialize_value() from values.py.

    - Helper: `_format_header(attrs: HeaderAttributes) -> str`: Format `[tag key=value ...]` back to text. Quote string values, leave numeric values unquoted.

    ```python
    @dataclass
    class ExtResource:
        """An [ext_resource] section."""
        type: str
        path: str
        id: str
        uid: str | None = None

    @dataclass
    class SubResource:

        type: str
        id: str
        properties: dict[str, Any]
        raw_section: Section | None = None  # For round-trip

    @dataclass
    class GdResource:
        """A parsed .tres file."""
        type: str
        format: int
        uid: str | None
        load_steps: int | None
        ext_resources: list[ExtResource]
        sub_resources: list[SubResource]
        resource_properties: dict[str, Any]
        _raw_header: HeaderAttributes | None = None
        _raw_sections: list[Section] | None = None

        def to_dict(self) -> dict:
            """Convert to JSON-serializable dict for resource inspect."""
            # ... include type, format, uid, ext_resources, sub_resources, properties
    ```

    - `serialize_tres(resource: GdResource) -> str`: If raw sections are available, delegate to `serialize_sections()` for round-trip fidelity. Otherwise, build sections from the data model.

    ```python
    @dataclass
    class SceneNode:

        connections: list[Connection]
    ```
