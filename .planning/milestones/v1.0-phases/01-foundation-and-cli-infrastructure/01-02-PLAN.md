    **Dataclasses (all frozen=True, slots=True for immutability and performance):**

    2. `Vector2i(x: int, y: int)` with same arithmetic but integer types. `to_godot()` returns `"Vector2i({x}, {y})"`.

    3. `Vector3(x: float, y: float, z: float)` with add, sub, mul, dot, length, cross. `to_godot()` returns `"Vector3({x}, {y}, {z})"`.

    4. `Vector3i(x: int, y: int, z: int)` with integer arithmetic.

    5. `Rect2(x: float, y: float, w: float, h: float)` with `contains(point: Vector2) -> bool`, `intersection(other: Rect2) -> Rect2 | None`, `size` property returning Vector2(w, h), `position` property returning Vector2(x, y). `to_godot()` returns `"Rect2({x}, {y}, {w}, {h})"`.

    6. `Rect2i(x: int, y: int, w: int, h: int)` with integer variants.

    7. `Color(r: float, g: float, b: float, a: float = 1.0)` with `to_godot()` returning `"Color({r}, {g}, {b}, {a})"`. Float formatting: use Godot's convention (no trailing zeros, but keep at least one decimal when the value is not whole).

    8. `Transform2D(xx: float, xy: float, yx: float, yy: float, ox: float, oy: float)` with `to_godot()` returning `"Transform2D({xx}, {xy}, {yx}, {yy}, {ox}, {oy})"`.

    9. `Transform3D` with 12 float fields (3x3 basis + 3 origin).

    10. `StringName(value: str)` with `to_godot()` returning `'&"{value}"'`.

    11. `NodePath(path: str)` with `to_godot()` returning `'NodePath("{path}")'`.

    12. `AABB(x: float, y: float, z: float, sx: float, sy: float, sz: float)`.

    13. `ExtResourceRef(id: str)` with `to_godot()` returning `'ExtResource("{id}")'`.

    14. `SubResourceRef(id: str)` with `to_godot()` returning `'SubResource("{id}")'`.

    **Float formatting helper:** Create a `_fmt_float(v: float) -> str` function that:
    - Returns `str(int(v))` if v is a whole number (e.g., 0.0 -> "0", 32.0 -> "32")
    - Returns `str(v)` otherwise (e.g., 1.5 -> "1.5")
    - This matches Godot's serialization behavior

    **Parse function:** `parse_value(text: str) -> Any`
    A function that takes a Godot value string and returns the corresponding Python type:
    - Detect constructor calls by finding the type name before `(`: "Vector2(", "Rect2(", etc.
    - For constructors: extract arguments between balanced parentheses, split by comma (respecting nested parens/strings), convert to appropriate types
    - `&"name"` -> StringName
    - `"string"` -> str (handle escaped quotes `\"`)
    - `null` -> None
    - `true` -> True, `false` -> False
    - Integer literals -> int
    - Float literals (containing `.` or `e`) -> float
    - `ExtResource("id")` -> ExtResourceRef
    - `SubResource("id")` -> SubResourceRef
    - `PackedStringArray(...)` -> list[str]
    - `PackedFloat32Array(...)` -> list[float]
    - `PackedInt32Array(...)` -> list[int]
    - `PackedByteArray(...)` -> bytes
    - For unknown constructors (per D-04 lenient parser): return the raw string

    **Serialize function:** `serialize_value(value: Any) -> str`
    Convert Python value back to Godot text format:
    - For dataclass types: call `.to_godot()`
    - For None: return `"null"`
    - For bool: return `"true"` or `"false"` (BEFORE int check since bool is subclass of int)
    - For int: return `str(value)`
    - For float: return formatted float string
    - For str: return `'"{escaped_value}"'` with `\"` escaping
    - For list: return `[{serialized_items}]`
    - For dict: return `{{serialized_key_value_pairs}}`

    **JSON serialization hook (D-03):** Create a `GodotJSONEncoder(json.JSONEncoder)` class that calls `serialize_value()` on Godot dataclass instances, producing Godot-native strings like `"Vector2(1.5, 2.0)"` in JSON output.
