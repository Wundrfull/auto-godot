    **Import guide help text** (SPRT-11, D-18): The `import_aseprite` command's help string should be a comprehensive multi-paragraph docstring that serves as the import guide. Include:

    ```
    Convert Aseprite JSON sprite sheet exports to Godot SpriteFrames .tres resources.

    Reads an Aseprite JSON metadata file (exported with --format json-array or
    --format json-hash) and generates a valid Godot SpriteFrames resource file.

    \b
    ASEPRITE EXPORT SETTINGS (recommended):
      aseprite -b input.ase --sheet sheet.png --data sheet.json \\
        --format json-array --sheet-type packed

    \b
    OPTIONS:
      --format json-array    (recommended, preserves frame order)
      --sheet-type packed    (or horizontal/vertical/rows/columns)
      --trim                 (optional; trimmed sprites are fully supported)
      --list-tags            (includes animation tags in JSON output)

    \b
    COMMON PITFALLS:
      1. Missing --list-tags: animations require frameTags in the JSON.
         Without --list-tags, all frames become a single "default" animation.
      2. Wrong --format: json-hash keys may reorder frames. Use json-array.
      3. Duplicate tag names: Godot requires unique animation names.

    \b
    EXAMPLES:
      gdauto sprite import-aseprite character.json
      gdauto sprite import-aseprite character.json -o sprites/character.tres
      gdauto sprite import-aseprite character.json --res-path res://art/character.png
    ```

    **Command definition:**

    ```python
    @sprite.command("import-aseprite")
    @click.argument("json_file", type=click.Path(exists=False))

    ```

    **Command body logic:**

    1. Resolve json_file to Path, check exists. If not, emit_error with code "FILE_NOT_FOUND" and fix "Check the path to your Aseprite JSON export".
    2. Call `parse_aseprite_json(json_path)`. Catch ValidationError and emit_error.
    3. Determine image_res_path:
       - If --res-path provided, use it directly
       - Otherwise: `"res://" + aseprite_data.meta.image` (per D-07: reference image in place)
    4. **Per-tag partial failure handling (per D-17):** Do NOT call `build_spriteframes()` as a single all-or-nothing operation. Instead, implement a per-tag iteration loop in the command that achieves D-17's partial failure semantics:

       ```python
       # Determine tags to process
       tags = aseprite_data.meta.frame_tags
       if not tags:
           tags = [AsepriteTag(name="default", from_frame=0,
                               to_frame=len(aseprite_data.frames) - 1,
                               direction=AniDirection.FORWARD, repeat=0)]

       # Create the shared ExtResource for the texture
       ext = ExtResource(type="Texture2D", path=image_res_path,
                         id=generate_resource_id("Texture2D"),
                         uid=uid_to_text(generate_uid()))

       successful_animations: list[dict] = []
       all_sub_resources: list[SubResource] = []
       warnings_list: list[str] = []

       for tag in tags:
           try:
               # Call a per-tag builder function (build_animation_for_tag
               # or _build_animation_for_tag from spriteframes.py)
               # that processes one tag and returns its SubResources + anim dict
               tag_subs, anim_dict = build_animation_for_tag(
                   tag, aseprite_data.frames, ext
               )
               all_sub_resources.extend(tag_subs)
               successful_animations.append(anim_dict)
           except (GdautoError, Exception) as exc:
               # Per D-17: warn on stderr, skip this tag, continue
               msg = f"Skipping animation '{tag.name}': {exc}"
               click.echo(msg, err=True)
               warnings_list.append(msg)

       if not successful_animations:
           # All tags failed; emit error and exit non-zero
           emit_error(GdautoError(
               message="All animation tags failed to process",
               code="SPRITE_ALL_TAGS_FAILED",
               fix="Check the Aseprite JSON for valid frame tags"
           ), ctx)
           ctx.exit(1)
           return

       # Assemble the GdResource from successful results
       load_steps = 1 + len(all_sub_resources) + 1
       resource = GdResource(
           type="SpriteFrames", format=3,
           uid=uid_to_text(generate_uid()),
           load_steps=load_steps,
           ext_resources=[ext],
           sub_resources=all_sub_resources,
           resource_properties={"animations": successful_animations}
       )
       ```

       This requires that Plan 01's `spriteframes.py` exposes a per-tag helper. Plan 01 Task 2 already defines the per-tag logic as part of `build_spriteframes`. The executor for Plan 01 MUST expose `build_animation_for_tag(tag: AsepriteTag, frames: list[AsepriteFrame], ext_resource: ExtResource) -> tuple[list[SubResource], dict[str, Any]]` as a public function (see revised Plan 01 Task 2 instruction). If `build_spriteframes` is also kept as a convenience wrapper, that is fine.

       If warnings_list is non-empty but some animations succeeded, the command exits with code 0 (per D-17: partial output with warnings).

    5. Determine output path (D-06):
       - If --output provided, use it
       - Otherwise: `json_path.with_suffix(".tres")`
    6. Call `serialize_tres_file(resource, output_path)`
    7. Emit result:
       - JSON mode: `emit({"output_path": str(output_path), "animation_count": len(successful_animations), "frame_count": len(all_sub_resources), "image_path": image_res_path, "warnings": warnings_list}, human_fn, ctx)`
       - Human mode: print summary like "Created character.tres with 3 animations (12 frames)"
       - If warnings_list: also print "Warning: {N} animation(s) skipped" in human mode

    **Human output function:**
    ```python
    def _print_import_result(data: dict[str, Any], verbose: bool = False) -> None:
        output = data["output_path"]
        anims = data["animation_count"]
        frames = data["frame_count"]
        click.echo(f"Created {output} with {anims} animation(s) ({frames} frames)")
        if data.get("warnings"):
            click.echo(f"  Warning: {len(data['warnings'])} animation(s) skipped due to errors", err=True)
        if verbose and "animations" in data:
            for anim in data["animations"]:
                click.echo(f"  {anim['name']}: {anim['frames']} frames, {anim['speed']} FPS")
    ```
