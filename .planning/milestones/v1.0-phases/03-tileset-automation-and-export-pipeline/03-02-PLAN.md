TileSet .tres per-tile terrain properties:
```
# Per-tile terrain assignment in TileSetAtlasSource sub-resource:
0:0/terrain_set = 0
0:0/terrain = 0
0:0/terrain_peering_bit/right_side = 0
0:0/terrain_peering_bit/bottom_right_corner = 0
0:0/terrain_peering_bit/bottom_side = 0
0:0/terrain_peering_bit/bottom_left_corner = 0
0:0/terrain_peering_bit/left_side = 0
0:0/terrain_peering_bit/top_left_corner = 0
0:0/terrain_peering_bit/top_side = 0
0:0/terrain_peering_bit/top_right_corner = 0
```

Resource-level terrain set declaration:
```
[resource]
terrain_set_0/mode = 0     # 0=MATCH_CORNERS_AND_SIDES, 1=MATCH_CORNERS, 2=MATCH_SIDES
terrain_set_0/terrains = [{"color": Color(1, 1, 1, 1), "name": "Terrain"}]
```

Physics polygon property:
```
0:0/physics_layer_0/polygon_0/points = PackedVector2Array(0, 0, 32, 0, 32, 32, 0, 32)
```

Peering bit names (Match Corners and Sides, 8 bits):
  right_side, bottom_right_corner, bottom_side, bottom_left_corner,
  left_side, top_left_corner, top_side, top_right_corner

Peering bit names (Match Sides only, 4 bits):
  right_side, bottom_side, left_side, top_side

Peering bit values:
  0 (or terrain_id >= 0) = expects neighbor with that terrain
  -1 = expects empty space
  omitted = unset (don't care)

    4. Define BLOB_47_LAYOUT: dict[tuple[int, int], dict[str, int]] with 47 entries.
       The standard blob-47 layout uses a grid where each (col, row) maps to a unique combination of the 8 peering bits. The layout follows the Tilesetter/tile_bit_tools convention:
       - The constraint is: if a side is -1 (empty), all corners adjacent to that side must also be -1.
       - This reduces 256 combinations (2^8) to exactly 47 unique tiles.
       - The standard arrangement is in a grid of approximately 12 columns x 4 rows (with some positions empty, using 47 of the available slots).
       - Use the community-standard mapping from the tile_bit_tools Godot plugin and the OpenGameArt blob tileset template.
       - Each entry: all 8 peering bit names as keys, values are 0 (terrain) or -1 (empty).

    5. Define MINIMAL_16_LAYOUT: dict[tuple[int, int], dict[str, int]] with 16 entries.
       Uses a 4x4 grid. Each entry has 4 side bits. The 16 combinations of 4 binary bits cover all cases:
       - (0,0): all sides -1 (isolated tile)
       - (1,0): right_side=0, others -1
       - ...through all 16 combinations in a standard marching-squares arrangement.

    6. Define RPGMAKER_LAYOUT: dict[tuple[int, int], dict[str, int]].
       RPG Maker uses a 2-column, 24-row arrangement (48 tiles) mapped to the blob-47 peering bit system with Match Corners and Sides mode. The mapping follows the community-documented RPG Maker A2 autotile convention translated to Godot peering bits.

    7. Define LAYOUT_MAP: dict[str, dict[tuple[int, int], dict[str, int]]] = {"blob-47": BLOB_47_LAYOUT, "minimal-16": MINIMAL_16_LAYOUT, "rpgmaker": RPGMAKER_LAYOUT}

    8. Function apply_terrain_to_atlas(atlas_sub: SubResource, layout: dict[tuple[int, int], dict[str, int]], terrain_set: int = 0, terrain_id: int = 0) -> None:
       For each (col, row), bits in layout.items():
         prefix = f"{col}:{row}"
         atlas_sub.properties[f"{prefix}/terrain_set"] = terrain_set
         atlas_sub.properties[f"{prefix}/terrain"] = terrain_id
         for bit_name, value in bits.items():
             atlas_sub.properties[f"{prefix}/terrain_peering_bit/{bit_name}"] = value

    9. Function add_terrain_set_to_resource(resource_props: dict[str, Any], layout_name: str, terrain_name: str = "Terrain") -> None:
       mode = TERRAIN_MODES[layout_name]
       resource_props["terrain_set_0/mode"] = mode
       resource_props["terrain_set_0/terrains"] = [{"color": Color(1, 1, 1, 1), "name": terrain_name}]

    1. Create src/gdauto/tileset/physics.py:

       Function parse_physics_rule(rule: str) -> tuple[range, str]:
         Parse "0-15:full" into (range(0, 16), "full").
         Accept formats: "N:type" (single tile), "N-M:type" (range).
         Valid types: "full", "none" (per D-04).
         Raise ValidationError with code INVALID_PHYSICS_RULE if malformed.

       Function apply_physics_to_atlas(atlas_sub: SubResource, rules: list[tuple[range, str]], columns: int, tile_width: int, tile_height: int) -> None:
         For each (tile_range, shape_type) in rules:
           For each index in tile_range:
             col = index % columns
             row = index // columns
             prefix = f"{col}:{row}"
             if shape_type == "full":
               # Full tile rectangle: 4 corners of the tile
               w, h = float(tile_width), float(tile_height)
               points = PackedVector2Array((0.0, 0.0, w, 0.0, w, h, 0.0, h))
               atlas_sub.properties[f"{prefix}/physics_layer_0/polygon_0/points"] = points
             elif shape_type == "none":
               # No collision: do not add any physics properties for this tile
               # (If properties existed from a prior run, this is a no-op since
               #  we build from scratch)
               pass
