    **Create src/gdauto/formats/project_cfg.py:**

    Data model:
    ```python
    @dataclass
    class ProjectConfig:
        """Parsed project.godot file."""
        global_keys: list[tuple[str, str]]  # Keys before any section (e.g., config_version=5)
        sections: dict[str, list[tuple[str, str]]]  # section_name -> ordered list of (key, raw_value)
        _raw_lines: list[str] | None = None  # Original lines for round-trip

        def get_global(self, key: str) -> str | None
        def get_value(self, section: str, key: str) -> str | None
        def section_names(self) -> list[str]
        def keys(self, section: str) -> list[str]
        def to_dict(self) -> dict  # JSON-serializable: {"global": {...}, "sections": {...}}
    ```

    **Parser implementation** (custom line-based, NOT configparser):

    `parse_project_config(text: str) -> ProjectConfig`:
    Line-by-line state machine. States: GLOBAL (before first section), SECTION (inside a section), MULTILINE (accumulating multi-line value).

    For each line:
    1. Strip trailing whitespace (preserve leading for indentation within multi-line values).
    2. If empty line: record for round-trip. Continue.
    3. If starts with `;`: comment line. Record for round-trip. Continue.
    4. If `[section_name]`: transition to that section. Record for round-trip.
    5. If `key=value` (first `=` splits key from value):
       - If in GLOBAL state: add to global_keys.
       - If in SECTION state: start property. Check bracket balance of value.
       - If brackets unbalanced: enter MULTILINE, accumulate.
    6. In MULTILINE: append line to buffer. Check bracket balance of accumulated text (with string-escape awareness). When balanced: finalize property value as the full accumulated string.

    Bracket depth tracking: same approach as common.py but can share or reimplement (project_cfg.py is a separate parser per research recommendation).

    `serialize_project_config(config: ProjectConfig) -> str`:
    If _raw_lines is available, return "\n".join(_raw_lines) + "\n" for perfect round-trip. Otherwise, reconstruct from data model: output global keys, then each section with its keys.

    **Important:** project_cfg.py stores all values as RAW STRINGS. It does not call parse_value() from values.py. The reason: project.godot values include Godot constructors like `PackedStringArray(...)` and `Object(...)` that are complex and context-dependent. The project commands layer (Plan 05) will interpret values as needed. This keeps the parser simple and forward-compatible (D-04).

    **Create src/gdauto/backend.py** (per D-17, D-20, PROJ-05, research Pattern):

    ```python
    @dataclass
    class GodotBackend:
        """Wrapper for Godot binary invocations."""
        binary_path: str | None = None
        timeout: int = 120
        _version: str | None = field(default=None, init=False, repr=False)
    ```

    Methods:

    - `__post_init__(self)`: If binary_path is None, do NOT discover yet (lazy). Discovery happens in ensure_binary().

    - `ensure_binary(self) -> str`: Discovery order per D-17: (1) self.binary_path if explicitly set (from --godot-path flag); (2) os.environ.get("GODOT_PATH") if set; (3) shutil.which("godot"). If none found, raise `GodotBinaryError(message="Godot binary not found", code="GODOT_NOT_FOUND", fix="Install Godot 4.5+ and add it to PATH, or set the GODOT_PATH environment variable, or pass --godot-path <path>")`. If found and `_version` is None, call `_check_version()` and cache result (per D-20). Return the binary path.

    - `_check_version(self) -> str`: Run `[binary, "--version"]` with timeout=10. Parse output with regex `r"(\d+)\.(\d+)"`. If major < 4 or (major == 4 and minor < 5), raise `GodotBinaryError(message=f"Godot {version} is too old (requires >= 4.5)", code="GODOT_VERSION_TOO_OLD", fix="Update Godot to version 4.5 or later")`. If parse fails, raise with code "GODOT_VERSION_PARSE". Return version string.

    - `run(self, args: list[str], project_path: Path | None = None, timeout: int | None = None) -> subprocess.CompletedProcess`: Call `ensure_binary()`. Build command: `[binary, "--headless"] + args`. If project_path, append `["--path", str(project_path)]`. Call `subprocess.run(cmd, capture_output=True, text=True, timeout=timeout or self.timeout)`. If returncode != 0, raise `GdautoError(message=..., code="GODOT_RUN_FAILED")` with stderr content. Return CompletedProcess.

    - `check_only(self, project_path: Path) -> subprocess.CompletedProcess`: Call `self.run(["--check-only"], project_path=project_path)`.

    - `import_resources(self, project_path: Path, quit_after: int = 30) -> subprocess.CompletedProcess`: Call `self.run(["--import", "--quit-after", str(quit_after)], project_path=project_path)`. Use --quit-after instead of --quit to avoid the race condition documented in GODOT-RESEARCH.md.
