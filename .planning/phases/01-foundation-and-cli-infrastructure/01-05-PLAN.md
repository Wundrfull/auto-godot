    **Subcommand: `project validate`** (per PROJ-02, PROJ-03):
    ```python
    @project.command()
    @click.argument("path", default=".", type=click.Path(exists=True))
    @click.option("--check-only", is_flag=True, help="Run Godot --check-only for script syntax validation")
    @click.pass_context
    def validate(ctx: click.Context, path: str, check_only: bool) -> None:
        """Validate project structure: check res:// paths, detect missing resources."""
    ```
    Implementation:
    1. Find and parse project.godot.
    2. Walk all .tscn and .tres files in the project directory.
    3. For each file, parse and extract all `res://` path references (from ext_resource path attributes and string values containing "res://").
    4. For each `res://` reference, resolve against project root (res:// maps to project root). Check if file exists. Collect missing_resources list.
    5. Collect orphan scripts: .gd files not referenced by any .tscn or .tres file (simple heuristic: walk all .gd files, check if their res:// path appears in any parsed file).
    6. Build report dict: { missing_resources: [...], broken_references: [...], orphan_scripts: [...], files_scanned: int, issues_found: int }.
    7. If --check-only flag is set: instantiate GodotBackend (using ctx.obj.godot_path), call backend.check_only(project_path). Parse stderr for script errors. Add to report under "script_errors" key. If Godot binary not available, the GodotBinaryError will be caught and reported as a warning (validate still succeeds for file-level checks).
    8. Call `emit(report, _display_validation_report_human, ctx)`.
    9. Exit with non-zero code if issues_found > 0.

    **Subcommand: `resource inspect`** (per FMT-07, D-03):
    ```python
    @resource.command()
    @click.argument("file", type=click.Path())
    @click.pass_context
    def inspect(ctx: click.Context, file: str) -> None:
        """Inspect a Godot resource (.tres) or scene (.tscn) file."""
    ```
    Implementation:
    1. Verify file exists. If not, raise ResourceNotFoundError with code "FILE_NOT_FOUND", fix "Check the file path and ensure the file exists".
    2. Determine file type by extension: `.tres` or `.tscn`. If neither, raise GdautoError with code "UNSUPPORTED_FORMAT", fix "resource inspect supports .tres and .tscn files".
    3. Parse the file:
       - .tres: call parse_tres_file(Path(file))
       - .tscn: call parse_tscn_file(Path(file))
    4. Build metadata wrapper (per research discretion recommendation):
       ```python
       data = {
           "file": str(Path(file).resolve()),
           "format": resource.format,
           "type": resource.type if hasattr(resource, 'type') else "Scene",
           "uid": resource.uid,
           "warnings": [],  # Populated if lenient parser encountered unknown content
           "resource": resource.to_dict(),
       }
       ```
    5. For JSON output: use `json.dumps(data, cls=GodotJSONEncoder, indent=2)` to serialize Godot value types as native strings per D-03.
    6. For human output: use rich.tree.Tree to display:
       - Root: file path and type
       - Branch: ext_resources (type, path, id for each)
       - Branch: sub_resources (type, id, key properties for each)
       - Branch: For .tres: resource properties. For .tscn: nodes (tree hierarchy based on parent), connections.
       - If verbose: show all property values, not just summaries.

    The human display function should use `rich.console.Console` for output (imported via rich-click's rich dependency). Use `rich.tree.Tree` for hierarchical display and `rich.table.Table` for tabular data (ext_resources list).
