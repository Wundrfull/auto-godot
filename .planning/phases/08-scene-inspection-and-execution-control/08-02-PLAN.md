<objective>
Implement the three scene inspection CLI commands: `debug tree` (SCENE-01), `debug get` (SCENE-02), and `debug output` (SCENE-03).

Purpose: These commands let users (and Claude Code agents) observe live game state without modifying it. `debug tree` provides the scene hierarchy (with optional extended metadata via `--full` per D-06), `debug get` reads individual node properties, and `debug output` captures game print() output and errors. Together they enable the "inspect" half of the deterministic testing pattern.

Output: New inspector.py module with async inspection functions, three new CLI subcommands in debug.py, all with tests.
</objective>

    Create `src/gdauto/debugger/inspector.py` with the following async inspection functions:

    1. **parse_scene_tree(data: list, offset: int = 0, parent_path: str = "") -> tuple[SceneNode, int]:**
       - Read 6 consecutive values from data starting at offset: child_count, name, type_name, instance_id, scene_file_path, view_flags
       - Compute path: `f"{parent_path}/{name}"` if parent_path else `f"/{name}"`
       - Advance offset by 6
       - Recursively parse child_count children, each returning (child_node, new_offset)
       - Return (SceneNode(..., children=children), final_offset)
       - Extended fields left at defaults (class_name=None, script_path=None, groups=[])
       - Source: godot-vscode-plugin helpers.ts parse_next_scene_node; Godot scene_debugger.h RemoteNode struct

    2. **parse_object_properties(data: list) -> tuple[int, str, list[NodeProperty]]:**
       - data[0] = object_id (int), data[1] = class_name (str), data[2] = flat property array
       - Parse property array in chunks of 6: name, type, hint, hint_string, usage, value
       - Skip entries where usage == 128 (category separators; these are editor UI sections, not real properties)
       - Return (object_id, class_name, list of NodeProperty)

    3. **async def enrich_scene_tree(session: DebugSession, root: SceneNode) -> SceneNode:**
       - Per D-06: populate extended metadata (class_name, script_path, groups) for each node via secondary inspect_objects calls
       - Walk the tree (iterative BFS or recursive DFS) and for each node:
         a. Call `await session.send_command("scene:inspect_objects", [[node.instance_id], False], response_key="scene:inspect_objects")`
         b. Parse response with parse_object_properties to get (obj_id, class_name, props)
         c. Set `node.class_name = class_name` if class_name differs from node.type_name (indicating a custom script class; if they match, leave as None since the base type is already in type_name)
         d. Find the "script" property in props; if present and its value has a resource_path attribute (GDScript resources), set `node.script_path = value.resource_path`. If no script property or no path, leave as None.
         e. Find any "groups" metadata or use a separate mechanism: Godot does not expose groups via inspect_objects directly. Groups can be obtained by checking if a "script" property exists and querying the node's metadata. For MVP, set groups=[] and document that groups require a different protocol approach. (Groups are not available via inspect_objects; they would require scene:get_node_groups or similar, which may not exist in the protocol. Leave as empty list with a TODO for future enhancement.)
       - Performance warning: This is O(n) network round-trips where n = total nodes. For large scenes (1000+ nodes), this can be slow. Document this tradeoff in CLI --help text for --full.
       - Return the root (modified in place)

    4. **async def get_scene_tree(session: DebugSession, max_depth: int | None = None, full: bool = False) -> SceneNode:**
       - Call `await session.send_command("scene:request_scene_tree", [], response_key="scene:scene_tree")`
       - Parse response with `parse_scene_tree(data, offset=0, parent_path="")`
       - If max_depth is not None, call `_prune_depth(root, max_depth)` per D-07
       - If full is True, call `await enrich_scene_tree(session, root)` per D-06
       - Return root SceneNode

    5. **_prune_depth(node: SceneNode, max_depth: int, current_depth: int = 0) -> SceneNode:**
       - If current_depth >= max_depth: set node.children = [] and return node
       - Otherwise recurse into children with current_depth + 1
       - Return node (mutates in place)

    6. **async def get_property(session: DebugSession, node_path: str, property_name: str) -> object:**
       - Call get_scene_tree(session) to get the full tree (no enrichment needed)
       - Call _find_node_by_path(tree, node_path) to locate the node
       - If node is None: raise DebuggerError(message=f"Node not found: {node_path}", code="DEBUG_NODE_NOT_FOUND", fix=f"Check that {node_path} exists in the running scene tree")
       - Call `await session.send_command("scene:inspect_objects", [[node.instance_id], False], response_key="scene:inspect_objects")`
       - Parse response with parse_object_properties(data)
       - Find property by name; if not found raise DebuggerError(code="DEBUG_PROPERTY_NOT_FOUND", fix=f"Check property name; use debug get --node {node_path} to list available properties")
       - Return property value

    7. **_find_node_by_path(root: SceneNode, target_path: str) -> SceneNode | None:**
       - If root.path == target_path: return root
       - Recursively search children
       - Return None if not found

    8. **format_output_messages(raw_output: list[list]) -> list[dict[str, str]]:**
       - Per Research Pitfall 7: each output entry is [[strings], [types]] where types 0=LOG, 1=ERROR, 2=LOG_RICH
       - Zip strings and types; map type 0 and 2 to "output", type 1 to "error"
       - Return list of {"text": str, "type": "output"|"error"} dicts
       - Per D-09 discretion: combine output and errors in single stream with type field

    9. **format_error_messages(raw_errors: list[list]) -> list[dict[str, str]]:**
       - Each error entry from Godot is [error, error_descr, source_file, source_line, source_func, warning, hr, min, sec, msec, callstack]
       - Return list of {"text": f"{error}: {error_descr}", "type": "error", "source": f"{source_file}:{source_line}"} dicts
